Слайд 2: 
Телемаркетинг - это использование телефона и современных средств связи для
для продажи услуг без последников.

Слайд 3:
Разработка осуществляется в ГК "ЦФТ" на базе ПЛ "Золотая Корона".
В компании уже существует ручной процесс телемаркетинга,
который необходимо автоматизировать.

Слайд 4:
На данном рисунке изображен существующий бизнес-процесс.
1) В самом начале, получаем клиентские заявки на займ от сервисов-партнером.
Это может быть приложение "Korona Pay" или какие-то магазины, в которых клиент
пытался получить займ;
2) Полученные заявки вручную анализируют бизнес-аналитики: создают фокус-группы, которые 
проходят через ручной отсев. Т.е они собирают данные из сервиса исторических данных, 
прогоняют через черные списки, получают доступные продукты. Вся собранная ифн. хранится 
в excel-документах;
3) Отфильтрованные заявки загружаются в БД системными администрами;
4) Далее главные операторы выгражают списки на обзвон и передают их операторам вручную;
5) Наконец, операторы производят звонки и фиксируют результаты вручную в excel-файлы.
Как видно, процесс достаточно сложный, много ручного труда, люди из разных команд,
что усложняет коммуникацию, разбор инцидентов и, в целом, с ростом нагрузки такой процесс
сложно масштабировать.

Слайд 5:
Для решения всех этих проблем в данной ВКР были поставлены цели:
1) Необходимо автоматизировать существующий процесс: 
от получения заявки до проведения звонка;
2) Произвести анализ существующих решений для построения телемаркетинга;
3) Разработать собственный сервис телемаркетинга с учетом потребностей бизнеса.

Слайд 6:
Одна из самых популярных и доступных систем - 1C:CRM.
Итог сравнения: 
1) позволяет управлять БД клиентов;
2) позволяет планировать звонки;
3) но нет встронной телефонии, поэтому нужно использовать доп. решения для звонков.

Слайд 7:
Еще существует MegaCRM.
Итог сравнения: 
1) позволяет управлять БД клиентов;
2) позволяет планировать звонки;
3) есть встроенная аналитика;
4) есть встроенная телефония.

Слайд 8:
Обе эти системы имеют недостатки:
1) Нет интеграций с сервисами "ЦФТ", 
что не дает автоматически фильтровать;
2) Нет возможности автоматически получать заявки, с помощью которых создаются
карточки клиента -> нельзя автоматически звонить;
3) Как итог, недостатки не позволяют использовать системы для
полной автоматизации существующего бизнес-процесса

Слайд 9:
Актуальность работы:
1) Есть бизнес-процесс, который необходимо автоматизировать;
2) Существующие системы не позволяют в полной мере покрыть потребности бизнеса.

Слайд 10:
В ГК "ЦФТ" есть готовое клиентское приложение и сервисы, с помощью которых
можно фильтровать клиентские заявки. Поэтому мне нужно было сосредоточиться только
на разработке серверного приложения и схеме хранения данных.

Слайд 11:
Всего есть три сервиса закрытых сервиса "ЦФТ", с которыми нужно интегрироваться.
Первый - черные списки. Это единая база нежелательных клиентов внутри группы компаний.
Там есть срок и причина блокировки клиента. Мы используем этот сервис для фильтрации
нежелательных клиентов. Поиск происходит по уникальному идентификатору устройства клиента или
по номеру телефона. Пример запроса на слайде.

Слайд 12:
Второй сервис - КАСПЕР. Это платформа учета большого объема данных. Это означает что там хранятся
все данные пользователя, который он оставил внутри ГК "ЦФТ". Это некий цифровой след.
На вход отправляем номер телефона клиента, уникальный идентификатор устройства, серию и номер паспорта клиента.
В ответ получаем большое количество клиентских данных.
Данные будем использовать для оценки пользователя с помощью сервиса АБРАМ.

Слайд 13:
АБРАМ - это ML-модели, обученные дата-аналитиками на основе признаков клиента.
В системе «Телемаркетинг» сервис используется для получения набора продуктов, которые в дальнейшем оператор будет предлагать клиенту.

Слайд 14:
Система была построена на основе предметно-ориентированного проектирования.
Т.е. в ней есть домен "займы", архитектурный слой "ядро" и архитектурный слой "интеграции".
1) В домене находятся специфичные для займов сервисы. Это значит что система может использоваться
не только для этого бизнесс-процесса, компании и т.д. Т.к. телемаркетинг это просто продажа услуг,
то достаточно переписать и мы можем использовать систему в какой-то другой предметной области.
2) В арх. слое "Ядро" находятся основные сервисы. 
Сервис кампаний - обрабатывает клиента в рамках его кампании, он создает задачи в рамках кампании,
создает задачи на обзвон и, в целом, проводит клиента по кампании.
Сервис АРМ служит входной точкой в систему для работы операторов. Доступ в АРМ осуществляется
с помощью сервиса аутентификации и авторизации.
3) Арх. слой ядро является адаптером сервиса телефонии. Для телефонии был выбран MightyCall.
Данный сервис создает задачи на обзвон, получает информацию о начале звонка.

Слайд 16:
Как только АРМ получает уведомление из брокера сообщений от адаптера телефонии о начале
звонка, он начинает подготавливать карточку клиента. Под карточкой клиента
подразумевается история прошлых коммуникаций с клиентом, список
доступных продуктов, личные данные клиента. Собранная карточка отдается
клиентскому приложению.

Слайд 17:
Финальной обработкой коммуникации является сохранение результата,
заполненного оператором в клиентском приложении.


Слайд 18:
Использовался язык Java, фреймворк Spring Boot, фрейморк Hibernate, брокер сообщений Kafka и субд PostgreSQL.

Слайд 19:
Как говорилось ранее, внутри ГК "ЦФТ" уже есть готовое клиенсткое приложение, поэтому тестирование разработанной системы
осуществлялось с помощью него.

В самом начале, оператору нужно войти в систему - ввести логин и пароль.

Слайд 20:
Как только оператор зашел, перед ним открывается список незавершенных коммуникаций. Это коммуникации, результаты которых не были заполнены.
На этой слайде у оператора нет незаполненных коммуникаций.

Слайд 21:
На этом есть несколько карточек, которые ожидают заполнения результата.
Кроме этого, на фоне запускается процесс опроса АРМ на наличие начала нового звонка.

Слайд 22:
Как только приложение получает положительный результат о начале звонка. Оно отправляет запрос в АРМ
для получения подготовленной карточки звонка. Здесь собрана вся информация о клиенте:
ФИО, номер телефона, гражданство, история прошлых коммуникаций, заявка на займ клиента, доступные продукты и т.д.

Слайд 25:
